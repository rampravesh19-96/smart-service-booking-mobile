# Feature Modules

This folder is reserved for feature-owned logic that grows beyond screen placeholders.

- `auth`: schemas, hooks, and auth-specific helpers
- `booking`: booking draft helpers, forms, and flow-specific domain logic
- `profile`: preferences, user settings, and support feature code

The current scaffold keeps screens in `src/screens` and shared pieces in `src/components`, while leaving this feature layer ready for Phase 2 implementation.
